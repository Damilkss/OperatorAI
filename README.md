## Проблема

### Коротко
> Рынок тарифов перенасыщен, а условия постоянно меняются. Пользователи тратят часы на ручное сравнение предложений операторов, открывая десятки вкладок, но всё равно переплачивают или сталкиваются с плохим качеством связи из-за неподходящего тарифа для их региона и стиля использования.

### Детально
Современный пользователь сталкивается с рядом трудностей при выборе тарифа:

1.  **Информационный шум.** Операторы предлагают запутанные линейки с мелким шрифтом и скрытыми условиями.
2.  **Региональные различия.** Цены, доступные тарифы и качество покрытия сильно отличаются в зависимости от региона, города и даже конкретного района.
3.  **Сложность сравнения.** Невозможно быстро и объективно сравнить десятки параметров (цена, минуты, гигабайты, скорость, пинг) у разных провайдеров.
4.  **Когнитивная нагрузка.** Пользователю приходится становиться 'экспертом' в тарифных линейках, вместо того чтобы просто пользоваться связью.
5.  **Страх ошибки.** Люди боятся подключить неподходящий тариф и либо переплачивают за ненужные услуги, либо сталкиваются с недостатком трафика или плохим покрытием.

**Наш сервис решает эту проблему**, переводя выбор из плоскости "сравни таблицы" в плоскость "опиши свои потребности". Мы подбираем тариф по смыслу, а не по названию.

## Целевая аудитория

Проект покрывает широкую аудиторию, но фокусируется на следующих ключевых сегментах:

### По типу услуги
1.  **🏠 Домашний интернет.** Люди, переезжающие в новую квартиру/дом, или те, кто недоволен текущим провайдером.
2.  **📱 Мобильная связь.** Все владельцы смартфонов, от школьников до пенсионеров.
3.  **📡 Мобильный интернет (4G/5G).** Жители частного сектора, фрилансеры, путешественники, которым нужен интернет "везде".

### По портрету пользователя
1.  **💰 Экономные (студенты, пенсионеры).** Ищут самый дешевый вариант с базовыми функциями (звонки, мессенджеры). Боятся скрытых платежей.
2.  **🚀 Активные пользователи (молодежь, офисные сотрудники).** Нужен безлимит или большой пакет для YouTube, TikTok, соцсетей, музыки.
3.  **🧭 Путешественники / Командировочные.** Важна стабильная связь и навигация в поездках по России, понимание региональных особенностей тарифов.
4.  **🎮 Геймеры.** Критически важны низкий пинг, стабильность соединения и высокая скорость для игр и стриминга.
5.  **👨‍👩‍👧 Семьи.** Ищут выгоду от подключения нескольких номеров (семейные тарифы) или пакетных предложений "Интернет + ТВ".
6.  **🏡 Владельцы загородной недвижимости.** Стоят перед выбором: проводить проводной интернет или использовать мобильного оператора с модемом.

## Стек технологий

Проект построен на легких, но эффективных технологиях, позволяющих запустить его даже на слабом сервере или локальной машине.

### Backend
-   **🐍 Python 3.10+** — основной язык разработки.
-   **🌶 Flask** — микро-веб-фреймворк для создания backend-логики и API. Выбран за простоту, гибкость и идеальную совместимость с NLP-библиотеками Python.
-   **📦 SQLite / PostgreSQL** — для хранения данных о тарифах, операторах, регионах и истории запросов. SQLite идеален для старта и разработки, PostgreSQL — для продакшена.

### Frontend
-   **HTML5 / CSS3** — базовая верстка.
-   **JavaScript (Vanilla)** — для интерактивности на клиенте (отправка запросов без перезагрузки страницы, анимации).
-   **Bootstrap / Tailwind CSS** — для быстрой и красивой стилизации интерфейса (на выбор).

### Инфраструктура
-   **Git** — контроль версий.
-   **Docker** (опционально) — для упаковки приложения со всеми зависимостями, включая NLP-модель.

## Локальная NLP-модель

Поскольку проект позиционируется как "подбор по описанию", нам нужна NLP-модель, способная понимать естественный язык. Мы не используем тяжелые облачные AI, а применяем легкую локальную модель, которая запускается прямо на сервере.

### Почему локальная модель?
-   🚀 **Скорость.** Мгновенные ответы без задержек на облачные API.
-   🔒 **Приватность.** Данные пользователей (их описания) не уходят в сторонние сервисы.
-   💰 **Экономия.** Нет платы за каждый запрос к API OpenAI и т.п.
-   🛠 **Простота развертывания.** Всё в одном месте.

### Выбор модели: `ruBERT-tiny2` или `ruRoberta-large` (в легком варианте)

Для нашей задачи идеально подходит модель семейства **BERT**, дообученная на русском языке. Мы будем использовать ее для **интент-анализа (intent classification)** и **извлечения ключевых параметров (NER)**.

-   **Модель:** `cointegrated/rubert-tiny2` (или `DeepPavlov/rubert-base-cased` для более точного, но чуть более тяжелого варианта).
-   **Тип:** Трансформер (архитектура BERT).
-   **Размер:** Менее 100 МБ (для tiny-версии), что позволяет запускать её даже на Raspberry Pi или самом дешевом VPS.
-   **Задача:** Классификация текста.

### Как модель обучается / работает
1.  **Разметка интентов (Целей пользователя).** Модель учится классифицировать запросы по типам:
    -   `home_internet` (проводной интернет в квартиру)
    -   `mobile_personal` (мобильная связь для себя)
    -   `mobile_family` (семейный тариф)
    -   `mobile_router` (симка для модема/роутера)
    -   `gaming` (игровой тариф с низким пингом)
    -   `social_only` (только соцсети и мессенджеры)

2.  **Извлечение параметров (NER).** Параллельно модель или дополнительные правила вытаскивают ключевые слова:
    -   Регион: *Москва, СПб, Казань*
    -   Потребности: *много гигабайт, безлимит, дешево, ТВ, YouTube*
    -   Ограничения: *в поездках, на даче*

3.  **Пример работы:**
    -   *Вход:* "ищу дешевую симку для интернета на даче в Подмосковье, буду смотреть ютуб и сидеть в телеграме"
    -   *Выход:* `intent: mobile_router`, `params: {region: "Подмосковье", price: "low", use_case: "youtube, telegram", location_type: "country_house"}`

### Библиотеки для работы с моделью
-   `transformers` (от Hugging Face) — для загрузки и работы с моделью.
-   `torch` (PyTorch) — бэкенд для вычислений (можно использовать легкую версию без CUDA, только CPU).
-   `razdel` / `nltk` — для предобработки текста (токенизация).

## Как это работает (MVP)

1.  **Пользователь заходит на сайт.**
2.  **Описывает свою ситуацию** в текстовом поле: *"Хочу интернет для квартиры в Москве, много играю и смотрю стримы"*.
3.  **Flask-бэкенд** принимает запрос и отправляет текст в нашу NLP-модель (`ruBERT-tiny2`).
4.  **Модель** определяет намерение (intent) и извлекает ключевые параметры (регион, потребности, бюджет).
5.  **Поисковый движок** (SQL-запросы с весами) ищет в базе данных тарифы, которые максимально соответствуют полученным параметрам, с учетом региона пользователя.
6.  **Пользователь получает** топ-3 наиболее подходящих тарифа с кратким объяснением, *почему они подходят именно ему*.
